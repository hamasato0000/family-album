## 1. 背景

近年、スマートフォンで撮影した子どもの写真や動画は大量に蓄積される一方、家族や親戚との共有はSNSやメッセージアプリに依存しがちである。  
これらの手段では以下の課題がある。

- プライバシーリスクが高い（SNSの公開範囲設定や誤送信の懸念）    
- 写真や動画が時系列・テーマ別に整理されにくい 
- 家族間でコメントや反応を集約できない

**ファミリーアルバム**は、この課題を解決するための、家族限定の安全な写真・動画共有プラットフォームを提供する。

## 2. 目的

- 子どもの成長記録を安全かつ手軽に家族間で共有できる仕組みを提供する。
- 家族や親戚がオンライン上で交流できる場をつくる。
- MVP段階では、最低限の共有・閲覧・コメント機能を実装し、ユーザー体験を確認する。

## 3. 対象ユーザー

- **管理者**：主に父母。アルバムの作成・管理を行う。
- **メンバー**：主に祖父母・叔父・叔母など。アルバムに招待され参加する。

## 4. 対象範囲（MVPで実装する機能）

### 4.1 ユーザー管理

- メールアドレス・パスワードによる会員登録・ログイン
- ログアウト
- パスワードリセット（メールリンク）

### 4.2 アルバム

- ユーザーは1つまで作成可能（その場合、自身が管理者となる）
- 招待を受けてメンバーとして参加する場合は複数参加可能（自身が管理者となっているアルバムを含めて最大10個まで）
- 招待はメールリンク経由で実施
- 管理者はアルバム内メンバーの権限設定を行える
- アルバム作成者＝管理者（変更不可）

### 4.3 コンテンツ（写真・動画）

- 管理者および権限付与されたメンバーがアップロード可能
- 対応形式：JPEG, PNG, MP4（仮）
- 最大ファイルサイズ：10MB（仮）
- 全ユーザーがアルバム内コンテンツを閲覧可能
- コメント投稿可能（最大500文字）（仮）
- 管理者および権限付与されたメンバーがコンテンツは削除可能
- 管理者およびコメント投稿者がコメントを削除可能
- 投稿者がコメントを編集可能

### 4.4 子ども情報

- 管理者のみ登録・編集可能
- 項目：名前／誕生日／性別
- 複数人登録可能

### 4.5 近況（履歴）

- アップロード・コメントの履歴表示（最新50件（仮））
- 全メンバーが閲覧可能
- 表示順：新しい順

## 5. 非対象範囲（MVPでは実装しない機能）

- ソーシャルログイン
- 多要素認証
- プッシュ通知
- 動画の自動変換や圧縮
- お気に入り機能
- ログイン履歴表示機能

## 6. 前提条件・制約

- 対応環境：スマートフォン・PCブラウザ（最新のChrome, Safari, Edge）
- オンライン接続必須
- HTTPS通信必須
- 保存データはクラウドストレージ利用（セキュリティ要件は別途設計書で定義）
